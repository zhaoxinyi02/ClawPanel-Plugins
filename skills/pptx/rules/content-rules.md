# 内容生成规则

根据用户需求生成结构化 PPT 内容时，按以下规则组织文档。生成结果写入 `output/content.md`，采用以下 Markdown 结构。

## 文档结构

```markdown
# 标题
主题的一句话概括

## 元信息
- 受众：
- 语言：中文/英文
- 预估页数：（默认 8-15 页，含封面/目录/致谢；用户未指定时不超过 15 页）

## 封面
- title: 主标题（≤80字）
- subtitle: 副标题（可选）
- description: 简短描述（可选）
- footer: 页脚（可选）
- **coverBackground**：**勿填**。封面背景使用模板默认，仅在用户明确要求「自定义封面图」时才填写 URL 或 output/images/xxx.png

## 目录
- title: Contents 或 目录
- intro: 目录前导语（可选）
- chapters: 章节列表，最多 5 项

## 正文（按章节组织）

### 章节 1
- 章节号: 01
- title: 章节标题
- subtitle: 英文/中文翻译（可选）
- statement: 章节金句（可选，用于 section-divider）
- **section-divider 右侧插图**：**尽量勿填**，使用模板默认。仅当用户明确要求更换章节配图时才填
- 内容块：按需选择以下类型

#### 类型A：左文右图 → `content-left-text-right-image`
- 主标题 + 副标题
- 3 组：小标题 + 正文（1-3行）
- 右侧图片：**必须**调用生图或搜图能力获取，根据本页内容生成/搜索贴合主题的配图

#### 类型B：双栏卡片 → `content-one-columns`
- 主标题 + 副标题
- 2 组：小标题 + 正文（3-5行），无图标

#### 类型C：双栏带图标 → `content-two-columns`
- 主标题 + 副标题
- 2 组：图标（`lucide:<icon-name>` 或配图 URL/路径） + 小标题 + 正文（1-3行）

#### 类型D：三栏带图标 → `content-three-columns`
- 主标题 + 副标题
- 3 组：图标（`lucide:<icon-name>` 或配图 URL/路径） + 小标题 + 正文（3-5行）

#### 类型E：四栏 2x2 → `content-four-columns`
- 主标题 + 副标题
- 4 组：图标（`lucide:<icon-name>` 或配图 URL/路径） + 小标题 + 正文（3-5行）

#### 类型F：数据高亮 → `data-highlight`
- 主标题 + 副标题
- 3 组：数字 + 描述（数字含单位，如 `8500万`、`3.75亿`、`+28%`，≤8字符）
- slogan: 底部标语（可选）

#### 类型G：金句页 → `quote`
- quote: 金句正文（**4-5行**，**必须可在一页内显示**，建议 **≤260 字符** / 中文 **≤140 字**；超出要主动删减或改写）
- subtitle: 翻译（可选）

#### 类型H：时间轴/流程 → `timeline`
- 主标题 + 副标题
- 3 步骤：编号（01/02/03） + 小标题 + 正文（2-4行）
- 适用于流程说明、实施步骤、时间线

#### 类型I：全图声明页 → `full-image`
- 全屏背景图：**必须**调用生图或搜图获取
- statement: 大字声明（≤120字符）
- subtitle: 补充说明（可选）
- tag: 右下角标签（可选，如 "Key Insight"、"2025 Vision"）

#### 类型J：对比页 → `comparison`
- 主标题 + 副标题
- 左栏（绿底）：标签 + 4 条对比项
- 右栏（灰底）：标签 + 4 条对比项
- 适用于 A vs B、Before/After、方案对比

## 致谢
- description: 底部描述（可选，默认 "Reach out to us if you have questions."）
```

## 内容量约束（防空防溢）

每页内容需**适中**：过少显得空荡，过多会溢出画板。按模板类型控制：

| 类型 | 每块正文 | 最少 | 最多 |
|------|----------|------|------|
| 左文右图 | block1–3Copy | 1 行 | 3 行 |
| 双栏卡片 | block1–2Copy | 3 行 | 5 行 |
| 双栏带图标 | block1–2Copy | 2 行 | 3 行 |
| 三栏带图标 | block1–3Copy | 2 行 | 4 行 |
| 四栏 2x2 | block1–4Copy | 2 行 | 3 行 |
| 金句页 | quote | 2 行 | 5 行（≤140 中文字 / ≤260 英文字符） |
| 时间轴 | step1–3Copy | 2 行 | 4 行 |
| 全图声明 | statement | 1 行 | 2 行（≤120 字符） |
| 对比页 | left/rightItem1–4 | 1 行 | 2 行 |

- **防空**：正文块不得仅 1 行或寥寥数字，需有实质说明
- **防溢**：超出最多行数时，拆成多页或删减，勿强行塞入一页

## 生成原则

1. **完整性**：封面、目录、正文、致谢缺一不可
2. **字体**：模板使用黑体字（苹方 PingFang SC / 微软雅黑 Microsoft YaHei），生成内容时无需指定字体
3. **适配性**：内容长度需匹配模板槽位（如 maxLength）及上表约束
4. **逻辑性**：章节顺序合理，目录与正文一致
5. **精简性**：每页信息量适中，避免堆砌
6. **语言一致**：全文统一中文或英文，subtitle 做互译
7. **按需选用模板**：根据内容量与结构选择模板，**同一模板可多次使用**，不要求每种模板各用一次
9. **页数控制**：用户未指定页数时，正文内容必须且一定控制在 **8-15 页**（含封面、目录、致谢），每章 2-3 页内容页即可，不要铺太多页
8. **内容量适中**：遵守上表最少/最多行数，**防空防溢**

## 示例（精简）

```markdown
# 产品发布会
Qoder Work 2024 产品发布

## 封面
- title: Qoder Work 2024
- subtitle: 智能开发新纪元

## 目录
- title: 目录
- intro: 本次发布涵盖以下内容
- chapters: [产品概览, 核心能力, 落地案例, 路线图]

## 正文
### 章节 1 - 产品概览
- 使用 左文右图
- 主标题: 产品概览
- 3 组内容 + 配图

### 章节 2 - 核心能力
- 使用 三栏带图标
- 3 个能力点 + 图标

...
```
