# 34ai-api-suite

## 用途
调用你提供的 34AI / Apifox 接口集合，统一处理小程序鉴权、抖音/小红书采集、数字人、AI 语音、AI 视频、AI 绘图、草稿剪辑、RPA 和自动剪辑等任务。

这份技能不是手工凭印象整理的。我已经逐条访问你给出的 Apifox `.md` 文档，并从每个文档里的 OpenAPI YAML 解析出接口方法、路径、参数位置、是否必填和部分返回字段。

- AI 语音相关：豆包文字生语音、index2 克隆音频、克隆结果查询
- AI 绘图相关：豆包文生图

## 配置项
在面板里安装技能后，先配置：

- `config.baseUrl`：你的 34AI API 网关地址
- `config.apiKey`：你的 API Key
- `config.authHeader`：API Key 放到哪个请求头里
- `config.authScheme`：如果要走 `Authorization` 头，是否加 `Bearer`
- `config.defaultTimeout`：请求超时时间，默认 30 秒

## 调用方式
这个技能适合按“接口名 + 参数”方式调用。

调用前先做这几步：

1. 读取技能配置中的 `config.baseUrl`
2. 读取技能配置中的 `config.apiKey`
3. 根据 `config.authHeader` 决定鉴权头写法：
   - `X-API-Key: <apiKey>`
   - 或 `Authorization: Bearer <apiKey>`
4. 再根据用户指定的接口名、路径、方法、参数发请求

推荐你在调用时明确给出：

1. 要调用的接口名称
2. 请求方法
3. 路径
4. query / body 参数
5. 你希望返回哪些字段

建议统一的调用描述格式：

```text
请调用 34AI API
- 接口：小程序获取access_token
- 方法：POST
- 路径：/cgi-bin/token
- query：grant_type=client_credential
- body / 参数：appid=..., secret=...
- 返回：access_token, expires_in
```

## 怎么用

### 用法 1：拿微信小程序 token
输入：

```text
请用 34AI API 技能获取小程序 access_token。
appid=你的appid
secret=你的小程序secret
```

### 用法 2：做抖音采集
输入：

```text
请调用 34AI API 的“抖音采集关键词文章(30s)”接口，关键词是“AI 编程”。
```

### 用法 3：做豆包文字生语音
输入：

```text
请调用 34AI API 的“豆包文字生语音”接口，把这段文案转成语音，并返回任务结果。
```

### 用法 4：做自动剪辑
输入：

```text
请调用 34AI API，先创建草稿，再添加视频和字幕，最后生成短视频。
```

## 输出建议
- 成功时返回核心结果字段，不堆整份大 JSON
- 异步任务接口要额外提示：
  - 任务 ID
  - 查询接口
  - 当前状态
- 失败时返回：
  - 接口名
  - HTTP 状态码
  - 错误码
  - 错误信息

## 注意事项
- 先确认 `baseUrl` 是真实可访问地址，不要直接使用文档示例域名
- `apiKey` 一律从技能配置读取
- 涉及批量采集、自动评论、发私信、发布视频等外发动作前，先让用户确认
- 对于异步接口，优先给出“发起任务 + 查询结果”两步式说明
- 如果用户只给了“接口名”但没给参数，先补齐最小必要参数再发请求
- 如果某接口返回任务 ID，不要假设它已经完成，要继续调用查询接口
